# Manus Outsourcing Workflow

> Объяснение на пальцах: что такое Manus, как с ним общаться, и как строить воркфлоу аутсорсинга для free-api-hunter.

## Что такое Manus?

Manus — это 4-5 джоберов на фрилансе. Каждый работает за кредиты (как зарплата), каждый умеет делать крутые штуки, но не умеет общаться с нами мгновенно — только через почту.

**Аналогия:** Ты отправляешь письмо с задачей → джобер читает → делает → присылает результат обратно. Это не чат в реальном времени, это task-based система.

## Как устроен Manus?

- **Sandbox (песочница):** отдельный компьютер, который включается когда приходит задача и выключается когда заканчивает. Не висит онлайн.
- **Кредиты:** валюта Manus. 300 кредитов в день на аккаунт. Простой вопрос — дешёвый. Презентация из 10 слайдов с исследованием — ~373 кредита.
- **Аккаунты:** у ЗавЛаба 4-5 аккаунтов = 1200-1500 кредитов/день суммарно.

## Как общаться с Manus (пошагово)

### Шаг 1: Отправить задачу (task.create)

```
POST /v2/task.create
Headers: x-manus-api-key: <ключ>
Body: { "message": { "content": "Описание задачи" } }
```

Response: `{ "task_id": "xxx", "task_url": "https://manus.im/app/xxx" }`

**Важно:** Чем конкретнее первое сообщение — тем лучше. Manus сам сделает всё внутри своей песочницы.

Плохо: *"Расскажи про API"*
Хорошо: *"Вот список из 20 API в JSON. Сделай таблицу с колонками: название, категория, лимит запросов, ссылка. Сохрани как XLSX"*

### Шаг 2: Дождаться результата (task.listMessages)

```
GET /v2/task.listMessages?task_id=xxx&order=desc&limit=5
```

Poll каждые 15-30 секунд. Manus работает асинхронно.

Статусы:
- `running` — работает
- `stopped` + `stop_reason: "finish"` — завершено
- `stopped` + `stop_reason: "ask"` — нужен ввод пользователя

### Шаг 3: Продолжить диалог (task.sendMessage) — если нужно

```
POST /v2/task.sendMessage
Body: { "task_id": "xxx", "message": { "content": "Поправь слайд 5" } }
```

Каждый task.sendMessage — это ещё одно письмо в ту же тему. Не новая сессия, а продолжение.

## Реальный пример из жизни

**Задача:** Сделать презентацию про AI в медицине.

1. Я отправила Manus письмо (task.create):
   > "Сделай презентацию про AI в медицине, 10 слайдов, с графиками"

2. Manus ответил: "Окей, делаю" (status: running)

3. Я подождала 20 секунд

4. Manus прислал готовый файл (status: stopped):
   - `AI Agents in Healthcare.pptx` (презентация)
   - `slide_notes.md` (заметки к слайдам)

**Стоимость:** 373 кредита.

## Воркфлоу аутсорсинга для free-api-hunter

**Сценарий:** Сделать отчёт о бесплатных API для медицины.

```
1. Наш агент (Dominika/Ворон) ищет 50 бесплатных медицинских API через сеть
2. Наш агент структурирует данные: названия, описания, лимиты, ссылки
3. Наш агент отправляет Manus:
   "Вот данные 50 API, сделай из них красивую презентацию на 15 слайдов
    с категориями, графиками рынка и рекомендациями"
4. Manus берёт данные, делает исследование, генерирует графики, собирает презентацию
5. Manus отдаёт нам файл PPTX + markdown с заметками
6. Мы забираем, проверяем, публикуем
```

**Принцип:** Мы делаем то, что умеем хорошо (поиск и сбор данных). Manus делает то, что умеет хорошо (оформление и генерация контента). Каждый на своём месте.

## Роутинг ключей

У нас 4-5 аккаунтов Manus. Когда нужно сделать задачу:

```
Проверяем баланс ключа 1 → хватает? → Используем
                                   → не хватает? → Ключ 2
Ключ 2 тоже пуст? → Ключ 3
```

Можно даже параллельно: 3 большие задачи одновременно на 3 разных аккаунта = в 3 раза быстрее.

## Правила эффективного общения

### Что делать:

- **Писать конкретно:** цель, ограничения, формат вывода — всё в первом сообщении
- **Использовать structured_output_schema** для гарантированного формата результата
- **Bulk вместо отдельных задач:** 10 элементов в одном task, не 10 отдельных
- **Использовать Projects** для повторяющихся задач (хранят инструкции и файлы)
- **Webhooks вместо polling** (task_stopped events)

### Чего не делать:

- **Не дёргать каждые 5 секунд** — Manus не онлайн-чат, он почта
- **Не разбивать одну задачу на много sendMessage** — Manus сам делает много шагов внутри
- **Не писать абстрактно** — "расскажи про API" приведёт к waiting states и потере кредитов

## Экономия кредитов

| Приём | Экономия |
|---|---|
| Конкретное первое сообщение | Меньше waiting states = меньше кредитов |
| Projects для повторяющихся задач | Не отправлять контекст заново |
| `lite` профиль для простых задач | Дешевле чем standard/max |
| "be concise", "stop after X" | Предотвращает лишние browsing cycles |
| Bulk: 10 элементов в одном task | Дешевле чем 10 отдельных tasks |

## Ограничения Manus

- Нет генерации видео
- Sandbox эфемерный (выключается после задачи)
- Нет доступа к GPU
- Нет постоянной памяти между task (каждый task = новый контекст)
- Временные ссылки на файлы (истекают)
